# AFL-1 Capstone Projects and Rubrics
## Three Portfolio Projects for Certification

**Version:** 1.0  
**Last Updated:** November 21, 2025  
**Weight:** 25% of final grade (40% of final assessment)

---

## Overview

Learners must complete THREE capstone projects demonstrating mastery of AFL-1 competencies:
1. **Personal Productivity System** - Individual efficiency improvement
2. **Professional Application** - Sector-specific problem solving
3. **Community Impact Project** - Social good application

Each project is evaluated using detailed rubrics with clear criteria.

---

## Capstone Project 1: Personal Productivity System

### Objective
Design and implement an AI-assisted productivity system that demonstrates **15-20% measurable improvement** in task completion time or quality.

### Requirements

**1. Baseline Documentation (Week 1)**
- Identify 5-10 repetitive tasks you perform regularly
- Measure current time spent on each task
- Assess current quality/effectiveness
- Document pain points and inefficiencies

**2. AI Integration Design (Week 2-3)**
- Select appropriate AI tools for each task
- Create optimized prompts for each workflow
- Design automation sequences
- Plan implementation timeline

**3. Implementation (Week 4-6)**
- Implement AI-assisted workflows
- Track time spent on each task
- Collect quality metrics
- Document challenges and solutions

**4. Results Analysis (Week 7)**
- Calculate time savings per task
- Assess quality improvements
- Measure overall productivity gain
- Calculate ROI (time saved vs. learning investment)

**5. Final Deliverable**
- **Written Report** (1500-2000 words):
  - Executive summary
  - Baseline vs. current comparison
  - Detailed workflow documentation
  - Results and analysis
  - Lessons learned
  - Future improvements
  
- **Prompt Library**:
  - 10+ optimized prompts used
  - Context and usage notes for each
  - Success rate data
  
- **Demo Video** (5-10 minutes):
  - Show before/after comparison
  - Demonstrate 2-3 key workflows
  - Explain results

### Evaluation Rubric (100 points)

#### 1. Baseline Documentation (15 points)
| Score | Criteria |
|-------|----------|
| 13-15 | Comprehensive baseline with 8+ tasks, detailed time tracking, quality metrics, clear pain points |
| 10-12 | Good baseline with 5-7 tasks, time tracking, some quality metrics |
| 7-9 | Basic baseline with 3-4 tasks, rough time estimates |
| 4-6 | Incomplete baseline, missing key data |
| 0-3 | Minimal or no baseline documentation |

#### 2. AI Integration Design (20 points)

#### 3. Implementation Quality (25 points)
| Score | Criteria |

| 16-20 | Strong implementation, most workflows working well, good documentation |

#### 4. Results and Impact (25 points)

## Capstone Project 2: Professional Application

### Objective
Solve a real-world problem from your professional sector using AI tools, demonstrating domain expertise and practical impact.

### Sector Options
Choose ONE sector that matches your profession:
1. **Agriculture** - Crop management, market analysis, farm planning
2. **Education** - Curriculum design, student assessment, learning materials
3. **MSMEs** - Business planning, marketing, customer service
4. **Healthcare** - Patient education, administrative efficiency, health information
5. **Governance** - Policy analysis, citizen communication, data reporting
6. **Financial Services** - Risk assessment, customer advisory, market analysis

### Requirements

**1. Problem Definition (Week 1)**
- Identify specific, real problem in your sector
- Explain current approach and limitations
- Define success criteria
- Estimate potential impact

**2. Solution Design (Week 2-3)**
- Research AI tools applicable to problem
- Design AI-assisted solution
- Create detailed implementation plan
- Identify risks and mitigation strategies

**3. Implementation (Week 4-6)**
- Build AI-assisted solution
- Test with real data/scenarios
- Iterate based on results
- Document process and learnings

**4. Validation (Week 7)**
- Measure solution effectiveness
- Compare to current approach
- Gather feedback from stakeholders
- Calculate impact metrics

**5. Final Deliverable**
- **Project Report** (2000-2500 words):
  - Problem statement and context
  - Solution design and rationale
  - Implementation details
  - Results and validation
  - Impact assessment
  - Recommendations for scaling
  
- **Technical Documentation**:
  - AI tools used and why
  - Prompt library (15+ prompts)
  - Workflow diagrams
  - Data/evidence of impact
  
- **Presentation** (10-15 slides):
  - Problem and solution overview
  - Key results and impact
  - Lessons learned
  - Future applications

## Capstone Project 3: Community Impact Project

### Objective
Use AI tools to create positive social impact in your community, demonstrating ethical AI usage and commitment to public good.

### Project Categories
Choose ONE category:
1. **Education Access** - Learning materials for underserved communities
2. **Health Awareness** - Public health information and resources
3. **Economic Empowerment** - Tools/resources for local businesses or job seekers
4. **Civic Engagement** - Information about rights, services, or participation
5. **Environmental Sustainability** - Resources for environmental awareness/action
6. **Accessibility** - Tools/content for people with disabilities

### Requirements

**1. Community Needs Assessment (Week 1)**
- Identify target community and their needs
- Research existing solutions and gaps
- Engage with community members
- Define project scope and goals

**2. Project Design (Week 2-3)**
- Design AI-assisted solution
- Ensure cultural appropriateness
- Plan for accessibility and inclusion
- Create implementation timeline

**3. Content/Tool Development (Week 4-6)**
- Create AI-assisted content or tools
- Test with community members
- Iterate based on feedback
- Ensure CACF compliance

**4. Deployment and Impact (Week 7)**
- Deploy to community
- Gather usage data and feedback
- Measure impact
- Plan for sustainability

**5. Final Deliverable**
- **Impact Report** (1500-2000 words):

  - Implementation process
  - Impact measurement
  - Community feedback
  - Sustainability plan
  
- **Project Outputs**:

- **Reflection Essay** (500 words):

## Submission Guidelines

### Timeline
- **Week 1-2:** Select projects and begin planning
- **Week 3-4:** Design and early implementation
- **Week 5-6:** Full implementation and testing
- **Week 7:** Finalize and document
- **Week 8:** Submit all three projects

### Submission Format
- **Digital Portfolio** containing:
  - All three project reports (PDF)
  - Technical documentation (PDF/Markdown)
  - Prompt libraries (Markdown/JSON)
  - Demo videos (MP4, max 50MB each)
  - Presentations (PDF/PPTX)
  - Evidence files (screenshots, data, feedback)

### Peer Review
- Each learner reviews 3 peers' projects
- Provides constructive feedback using rubrics
- Scores are advisory (instructor has final say)

### Revision Policy
- One revision allowed per project
- Must be submitted within 2 weeks of feedback
- Revised projects can achieve full points

## Overall Capstone Scoring

**Total Points:** 300 (100 per project)

**Grade Calculation:**
- **Excellent (90-100%):** 270-300 points - Demonstrates mastery
- **Good (80-89%):** 240-269 points - Strong competence
- **Satisfactory (70-79%):** 210-239 points - Adequate competence (PASS)
- **Needs Improvement (<70%):** <210 points - Requires revision

**Certification Requirement:**
- Minimum 210/300 points (70%)
- All three projects must score 70+ individually
- No project can be skipped
