# AFL-0 Practical Tasks and Assessment Rubrics

**Version:** 2.0  
**Last Updated:** November 21, 2025  
**Total Tasks:** 10 hands-on exercises + 3 final assessments  
**Pass Threshold:** 70% average across all tasks

## Part 1: Practical Tasks (10 Tasks)

### Task 1: Voice Assistant Interaction
**Duration:** 15 minutes  
**Objective:** Demonstrate basic voice assistant usage

**Instructions:**
1. Use a voice assistant (Google Assistant, Alexa, or Siri) in Hindi or your regional language
2. Complete 5 different tasks:
   - Ask for weather in your city
   - Set an alarm or reminder
   - Translate a phrase to another language
   - Ask a factual question (e.g., "Who is the Prime Minister?")
   - Play music or news
3. Document: Take screenshots or note responses
4. Reflect: Which tasks worked well? Any errors?

**Submission:**
- Screenshots of 5 interactions OR
- Written log of commands and responses
- Brief reflection (100 words)

**Rubric:**
| Criterion | Excellent (4) | Good (3) | Satisfactory (2) | Needs Improvement (1) |
|-----------|---------------|----------|------------------|----------------------|
| **Task Completion** | All 5 tasks completed successfully | 4 tasks completed | 3 tasks completed | < 3 tasks completed |
| **Language Use** | Used regional language/Hindi effectively | Mixed language use | Mostly English | Only English |
| **Documentation** | Clear screenshots/logs for all tasks | Most tasks documented | Some documentation | Minimal documentation |
| **Reflection** | Insightful analysis of experience | Good reflection | Basic reflection | Minimal reflection |

**Passing Score:** 8/16 points (50%)

### Task 2: UPI Privacy Settings Review
**Duration:** 20 minutes  
**Objective:** Understand and adjust privacy settings

**Instructions:**
1. Open your UPI app (PhonePe, Google Pay, Paytm, or any UPI app)
2. Navigate to Settings → Privacy/Permissions
3. Review what permissions the app has:
   - Contacts access
   - Location access
   - SMS access
   - Camera access
   - Storage access
4. For each permission, determine if it's necessary
5. Adjust settings to minimize data sharing while maintaining functionality
6. Document your changes

**Submission:**
- Before and after screenshots of permission settings
- Table listing each permission, whether you kept/disabled it, and why
- Reflection on privacy vs. functionality trade-offs (150 words)

**Rubric:**
| Criterion | Excellent (4) | Good (3) | Satisfactory (2) | Needs Improvement (1) |
|-----------|---------------|----------|------------------|----------------------|
| **Permission Review** | All permissions reviewed and justified | Most permissions reviewed | Some permissions reviewed | Minimal review |
| **Privacy Awareness** | Demonstrates strong privacy understanding | Good privacy awareness | Basic privacy awareness | Limited awareness |
| **Documentation** | Complete before/after screenshots + detailed table | Good documentation | Basic documentation | Minimal documentation |
| **Reflection** | Thoughtful analysis of trade-offs | Good reflection | Basic reflection | Minimal reflection |

**Passing Score:** 8/16 points (50%)

### Task 3: Fact-Checking AI Content
**Duration:** 30 minutes  
**Objective:** Verify AI-generated information

**Instructions:**
1. Ask an AI tool (ChatGPT, Google Bard, or any AI assistant) a factual question about India (e.g., "What are the benefits of PM-KISAN scheme?")
2. Get the AI response
3. Fact-check the response using at least 3 reliable sources:
   - Official government website
   - Reputable news outlet
   - Academic or expert source
4. Identify: What's accurate? What's missing? Any errors?
5. Document your findings

**Submission:**
- AI response (screenshot or text)
- List of 3 sources used for verification (with URLs)
- Fact-check analysis: Accurate points, errors, missing information
- Conclusion: Is the AI response reliable? (200 words)

**Rubric:**
| Criterion | Excellent (4) | Good (3) | Satisfactory (2) | Needs Improvement (1) |
|-----------|---------------|----------|------------------|----------------------|
| **Source Quality** | 3 highly reliable sources | 2-3 reliable sources | 1-2 sources | Unreliable sources |
| **Fact-Checking** | Thorough verification of all claims | Good verification | Basic verification | Minimal verification |
| **Analysis** | Detailed identification of accurate/inaccurate/missing info | Good analysis | Basic analysis | Minimal analysis |
| **Conclusion** | Well-reasoned assessment of reliability | Good conclusion | Basic conclusion | Weak conclusion |

**Passing Score:** 8/16 points (50%)

### Task 4: Identifying AI Bias
**Duration:** 30 minutes  
**Objective:** Recognize bias in AI outputs

**Instructions:**
1. Use an AI tool to generate content for 3 different scenarios:
   - Job description for "software engineer"
   - Job description for "nurse"
   - Loan eligibility criteria
2. Analyze each output for potential bias:
   - Gender bias (language suggesting one gender over another)
   - Regional bias (favoring certain locations)
   - Educational bias (requiring specific institutions)
   - Any other stereotypes
3. Document biases found
4. Suggest how to make the content more fair and inclusive

**Submission:**
- AI-generated content for all 3 scenarios
- Bias analysis for each (identify specific biased language/criteria)
- Suggestions for fairer alternatives
- Reflection on why AI might have these biases (200 words)

**Rubric:**
| Criterion | Excellent (4) | Good (3) | Satisfactory (2) | Needs Improvement (1) |
|-----------|---------------|----------|------------------|----------------------|
| **Bias Identification** | Identifies multiple biases with specific examples | Identifies some biases | Identifies few biases | Minimal bias identification |
| **Analysis Depth** | Deep understanding of bias sources and impacts | Good analysis | Basic analysis | Superficial analysis |
| **Fair Alternatives** | Thoughtful, inclusive suggestions for all scenarios | Good suggestions | Basic suggestions | Weak suggestions |
| **Reflection** | Insightful understanding of AI bias origins | Good reflection | Basic reflection | Minimal reflection |

**Passing Score:** 8/16 points (50%)

### Task 8: CACF Grievance Filing (Practice)
**Duration:** 30 minutes  
**Objective:** Understand grievance redress process

**Instructions:**
1. Choose a hypothetical CACF violation scenario (or use real if applicable):
   - AI hiring tool rejected you unfairly
   - AI loan system discriminated based on region
   - AI app violated your privacy
   - AI service not available in your language
2. Document the violation:
   - What happened?
   - Which CACF article was violated?
   - Evidence you would collect
3. Draft a CACF complaint (practice, don't submit unless real)
4. Outline the 3-tier escalation process

**Submission:**
- Scenario description
- CACF article violated and why
- Evidence list (screenshots, documents, witnesses)
- Draft complaint (200-300 words) including:
  - Your details
  - AI system details
  - Violation description
  - Relief sought
- 3-tier escalation plan

**Rubric:**
| Criterion | Excellent (4) | Good (3) | Satisfactory (2) | Needs Improvement (1) |
|-----------|---------------|----------|------------------|----------------------|
| **CACF Understanding** | Correctly identifies article and violation | Good understanding | Basic understanding | Misunderstands CACF |
| **Evidence Collection** | Comprehensive, relevant evidence | Good evidence | Some evidence | Minimal evidence |
| **Complaint Quality** | Well-written, clear, specific | Good complaint | Basic complaint | Weak complaint |
| **Process Knowledge** | Clear understanding of 3-tier system | Good knowledge | Basic knowledge | Confused about process |

**Passing Score:** 8/16 points (50%)

### Task 10: AI Policy Participation
**Duration:** 45 minutes  
**Objective:** Engage in democratic AI governance

**Instructions:**
Choose ONE of the following activities:

**Option A: Write to Representative**
- Write a letter to your MP/MLA about an AI-related issue
- Issue could be: AI bias, language access, privacy, jobs, education
- Include: Problem description, impact on community, specific request

**Option B: Public Consultation**
- Find an ongoing AI policy consultation (MyGov or government website)
- Read the draft policy/proposal
- Submit written feedback with your opinion and suggestions

**Option C: Community Discussion**
- Organize or participate in community discussion on AI
- Topic: Impact of AI on your community (jobs, services, rights)
- Document discussion points and outcomes

**Submission:**
- Copy of letter/feedback/discussion notes
- Description of issue and your position
- Action taken (sent letter, submitted feedback, held discussion)
- Reflection on civic engagement experience (200 words)

**Rubric:**
| Criterion | Excellent (4) | Good (3) | Satisfactory (2) | Needs Improvement (1) |
|-----------|---------------|----------|------------------|----------------------|
| **Issue Understanding** | Deep understanding of AI policy issue | Good understanding | Basic understanding | Superficial understanding |
| **Argument Quality** | Well-reasoned, specific, actionable | Good arguments | Basic arguments | Weak arguments |
| **Civic Action** | Meaningful engagement with process | Good engagement | Basic engagement | Minimal engagement |
| **Reflection** | Thoughtful analysis of democratic participation | Good reflection | Basic reflection | Minimal reflection |

## Part 2: Final Assessment Rubrics

### Final Assessment 1: Practical Demonstration (15% of total grade)

**Task:** Use AI tools to solve a real problem from your life

**Duration:** 1-2 hours  
**Submission:** Video (5-10 min) OR Written report (500-750 words) + Screenshots

**Requirements:**
1. Identify a real problem you face (work, study, home, community)
2. Select appropriate AI tools to address it
3. Apply AI tools systematically
4. Document the process and outcome
5. Reflect on effectiveness

**Examples:**
- Use AI to plan family budget and reduce expenses
- Use AI to improve farming decisions (crop selection, market prices)
- Use AI to create teaching materials for your students
- Use AI to research and write a community petition
- Use AI to start or improve a small business

**Rubric:**

| Criterion | Excellent (9-10) | Good (7-8) | Satisfactory (5-6) | Needs Improvement (1-4) |
|-----------|------------------|------------|-------------------|------------------------|
| **Problem Relevance** (10 points) | Significant, real problem clearly defined | Real problem, well-defined | Somewhat relevant problem | Trivial or unclear problem |
| **AI Tool Selection** (10 points) | Optimal tool choice, well-justified | Good tool choice | Acceptable tool choice | Poor tool choice |
| **Application Process** (20 points) | Systematic, effective use of AI | Good AI application | Basic AI use | Ineffective AI use |
| **Outcome** (20 points) | Problem significantly improved/solved | Good improvement | Some improvement | Minimal improvement |
| **Documentation** (20 points) | Comprehensive, clear documentation | Good documentation | Adequate documentation | Poor documentation |
| **Reflection** (20 points) | Deep insights on AI effectiveness, limitations, learning | Good reflection | Basic reflection | Minimal reflection |

**Total:** 100 points  
**Passing Score:** 70 points  
**Weight:** 15% of final grade

### Final Assessment 2: Critical Analysis (10% of total grade)

**Task:** Evaluate AI-generated content for reliability and bias

**Duration:** 45 minutes  
**Submission:** Written analysis (400-600 words)

**Provided Materials:**
- AI-generated content (news summary, health advice, financial recommendation, or policy analysis)
- Learners must evaluate without prior knowledge of accuracy

**Requirements:**
1. Fact-check all claims using reliable sources
2. Identify any biases (gender, regional, caste, language, etc.)
3. Assess overall quality and reliability
4. Provide recommendations for improvement

**Rubric:**

| Criterion | Excellent (9-10) | Good (7-8) | Satisfactory (5-6) | Needs Improvement (1-4) |
|-----------|------------------|------------|-------------------|------------------------|
| **Fact-Checking** (25 points) | All claims verified with multiple reliable sources | Most claims verified | Some verification | Minimal verification |
| **Bias Identification** (25 points) | Identifies multiple biases with specific examples | Identifies some biases | Identifies few biases | Misses obvious biases |
| **Source Quality** (15 points) | Uses highly authoritative sources | Uses reliable sources | Uses acceptable sources | Uses questionable sources |
| **Analysis Depth** (20 points) | Nuanced, comprehensive analysis | Good analysis | Basic analysis | Superficial analysis |
| **Recommendations** (15 points) | Specific, actionable improvements | Good recommendations | Basic recommendations | Vague recommendations |

**Total:** 100 points  
**Passing Score:** 70 points  
**Weight:** 10% of final grade

## Grading Summary

### Continuous Assessment (70%)
- Unit Quizzes (20%): 5 quizzes × 4% each
- Practical Tasks (30%): 10 tasks × 3% each
- Reflection Journal (20%): Cumulative across all units

### Final Assessment (30%)
- Practical Demonstration (15%)
- Critical Analysis (10%)
- Civic Engagement Project (5%)

### Overall Passing Criteria
- **Minimum 70% overall**
- **Minimum 70% on each unit quiz**
- **Minimum 70% on final assessments**
- **Completion of all required tasks**

## Implementation Guidelines

### For Facilitators

**Grading Practical Tasks:**
1. Use rubrics consistently
2. Provide constructive feedback
3. Allow resubmission if < 70%
4. Focus on learning, not gatekeeping

**Supporting Learners:**
1. Provide examples and templates
2. Offer extra help for struggling learners
3. Celebrate successes
4. Create safe space for experimentation

**Quality Assurance:**
1. Calibrate grading with other facilitators
2. Document common issues
3. Share best practices
4. Continuously improve rubrics

### For Learners

**Completing Tasks:**
1. Read instructions carefully
2. Ask for clarification if needed
3. Document your work thoroughly
4. Reflect honestly on learning

**Improving Scores:**
1. Review rubrics before starting
2. Use feedback to improve
3. Resubmit if below 70%
4. Learn from mistakes

**Getting Help:**
1. Contact facilitator
2. Join study groups
3. Use online resources
4. Practice with peers
